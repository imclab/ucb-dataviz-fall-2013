Getting started with the R console

You should have R installed on your machine or desktop. If you don't, install it and run it form the applications menu. You should see the R console, which looks like this:

Before we start breaking things, let's get our workspace set up properly. Make a folder on your desktop called cuny-tutorial. In that folder, make a new file called tutorials.R in a text editor and save it to your project folder. Start by typing your working directory in your R text file and pasting it into the R console. (Cutting and pasting from static files is not the classiest way to do things, since there's an R IDE now, but this is how the Godfather does it, so it's good enough for us.)

  
    setwd("~/Desktop/cuny-tutorial")

Start typing things into the console and discuss what happens after each.

  
    "I should exercise more."

    1+1

    1==2

    1:1000

    animals <- c("bears","monkeys","donkeys")

    animals[1]

    class(animals)

    class(1==2)

    class(1:1000)
Questions/thoughts?

R isn't hard, but it can feel difficult because it's picky. The difference between a map or chart that works and one that's broken can sometimes just be a missing argument to a method (like stringsAsFactors=F or horiz=T), which can be frustrating. Rather than try to cover all of thes eexhaustively, we'll just start making some projects and learn as we go.
